# 로컬 생성 환경 벤치마크 — 아침 보고서

> 밤샘 자동 실행 결과. Intel Arc iGPU(내장) + PyTorch XPU 환경.
> 목적: **"이 PC로 계속 갈지, 더 좋은 PC로 갈아탈지"** 판단 근거 확보.

---

## 1. 하드웨어

```
CPU : Intel Core Ultra 7 155H
GPU : Intel Arc Graphics (내장, 디스크리트 없음)
RAM : 31.6GB (iGPU가 시스템 RAM 공유)
```

핵심 특성: **iGPU가 시스템 RAM을 통째로 공유** → 용량 부족(OOM)은 잘 안 나지만, 메모리 대역폭 탓에 **스텝당 연산이 느리다.**

---

## 2. 속도 실측 (전부 스크립트 타이머 기준)

| 작업 | 스텝 | 해상도 | 시간 | 스텝당 |
|---|---|---|---|---|
| SDXL base | 20 | 1024 | **149.7초** (재측정 중앙값) | 7.5초 |
| SDXL base | 20 | 1024 | 149.3초 (첫 측정) | 7.5초 |
| Juggernaut 25스텝 | 25 | 1024 | 213.2초 | 8.5초 |
| **Lightning 4스텝** | 4 | 1024 | **30.5초** | 7.6초 |
| Lightning 8스텝 | 8 | 1024 | 47.0초 | 5.9초 |
| Lightning 4스텝 | 4 | 768 | 16.8초 | 4.2초 |
| **Juggernaut + Lightning 8스텝** | 8 | 1024 | **43.4초** | — |
| **SVD 영상 (14프레임)** | 25 | 1024×576 | **1541.7초 (25.7분)** | 프레임당 110초 |

### 중요: "진행바 착시" 기록
생성 중 tqdm 진행바가 한때 **1.98 it/s(약 12초)**로 표시돼 "15배 빨라졌나?" 착각했으나,
**스크립트 타이머 실측은 213초**였다. 원인은 **XPU 비동기 실행** — 연산을 큐에 넣는 속도로
진행바가 앞서 달리고, 실제 계산은 마지막 동기화(VAE 디코드) 때 몰린다.
→ **XPU에서는 진행바 it/s를 믿지 말 것. 벽시계 타이머만 신뢰.**
캐시 데워진 상태 3회 재측정도 149.7초로 동일 → **첫 컴파일 오염 아님, 이게 진짜 속도.**

모든 조건이 **스텝당 7~8.5초**로 일관. Lightning이 빠른 건 하드웨어가 아니라 **스텝 수를 4~8개로 줄이기 때문.**

## 3. 화질 판정

### 이미지 — 합격
- **Juggernaut 25스텝** (`예인_Juggernaut_25스텝_고화질.png`): 피부 질감·모공·필름 그레인까지 "실제 필름 사진" 수준. 이게 목표 화질.
- **Juggernaut + Lightning 8스텝** (43초): 25스텝(213초) 화질을 **거의 그대로 유지.** → 이게 실전 워크플로우.
- Lightning 4스텝(30초): 잔디테일 약간 거칠지만 실용 가능. 컷당 30초라 재추첨 부담 없음.

### 발견된 문제 — 프롬프트 누출 (해결됨)
`예인_..._늑대니트.png` 참고: 스웨터에 **늑대 얼굴이 짜여 들어감.**
원인 = 프롬프트의 `wolf-cut`(울프컷) + `puppy-like`(강아지상) 토큰이 **머리/얼굴이 아니라 옷 그래픽으로 새어나감.**
base Lightning에서 보이던 "니트 가슴께 얼룩"의 정체가 이것.

**해결책** (다음 생성부터 적용):
- `wolf-cut` → `shaggy layered bob`
- `puppy-like features`는 얼굴 근처에만 배치하거나 제거
- 네거티브에 `graphic print on clothing, animal on sweater` 추가

### 영상 (SVD) — 화질/움직임 합격, 속도 탈락
`예인_SVD_모션_첫중간끝.png` / `예인_SVD_영상테스트_2초.mp4` 참고:
- 은은한 카메라 푸시인 + 고개 미세 회전. **정지 사진이 아니라 살아있는 초상.**
- 인물 정체성 안 무너짐(얼굴 안 녹음). 머리카락·보케 자연스럽게 흐름.
- **그록이 못 하던 "내 이미지에서 시작"을 SVD는 해냄** (첫 프레임 고정).
- 처음에 원하던 "잔잔한 루프 무드 영상" 바로 그 느낌.
- **단, 2초 클립 하나에 25.7분.** OOM은 안 나지만 실용성 없음.

## 4. 결론

| 작업 | 이 PC | 판정 |
|---|---|---|
| 이미지 (Lightning 4~8스텝) | 30~47초 | ✅ 실용 |
| 이미지 (Juggernaut 실사, L8) | 43초 | ✅ 실용 — 목표 화질 |
| 이미지 (Juggernaut 25스텝 최고화질) | 213초 | △ 배치로 가능 |
| **영상 (SVD 로컬 AI 모션)** | **25.7분/2초** | ❌ 비실용 → GPU PC 필요 |

### 권고
- **이미지 생산은 지금 이 PC로 충분.** Juggernaut + Lightning 8스텝(43초)이 실사 화질 + 실용 속도를 동시에 잡음. MV 스틸 30장 = 약 20분(배치).
- **로컬 AI 영상(SVD·Wan·립싱크)은 이 PC로 불가.** 순수 속도 문제이며 GPU PC로만 해결.
- **따라서**: 곡·스틸·MV 조립(하드컷)·채널 운영까지는 **지금 PC로 즉시 시작 가능.** GPU PC는 "인물이 실제로 움직이는 영상"이 파이프라인의 병목이 되는 시점에 결정해도 늦지 않음.
- SVD가 화질·움직임에서 합격점을 준 건 의미가 큼 — **GPU PC로 갈아타면 로컬 영상이 실제로 성립한다**는 증거.
