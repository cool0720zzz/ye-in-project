# 예인 영상제작 기술 가이드 — 인물 모션 구현편 (v2)

> **이 문서는 Claude Code 인수인계용 + 직접 제작 시 참고 문서입니다.**
> 정지 이미지에서 인물의 움직임, 표정 변화, 립싱크를 구현하는 방법과
> 상업적 이용 가능 여부, 실제 제작 순서를 정리합니다.

---

## 0. 핵심 전략: "2시간 중 실제 모션이 필요한 건 약 2분 30초"

| 장면 유형 | 필요 시간 | 모션 수준 | 구현 방법 |
|-----------|----------|----------|-----------|
| 인트로 (착석/마이크세팅/인사) | ~8초 × 1회 | **실제 모션** | AI 영상 생성 |
| 허밍 + 눈 감기 (곡 시작) | ~3초 × 30곡 = ~90초 | **표정 변화** | AI 영상 생성 |
| 노래 중 정지 | ~2.5분 × 30곡 = ~75분 | 미세한 움직임 | 코드 (Breathing + Ken Burns) |
| "쉬어갈게요" 멘트 + 퇴장 | ~4초 × 4회 = ~16초 | **실제 모션** | AI 영상 생성 / SadTalker |
| 재착석 | ~3초 × 4회 = ~12초 | **실제 모션** | AI 영상 생성 |
| 인스트루멘탈 배경 | ~24분 | 없음 | 정지 이미지 + 코드 효과 |

**AI 모션 생성 필요: 약 2분 30초 (유니크 클립 8~12개)**
클립 재사용(허밍 3종 로테이션, 퇴장/착석 1~2종 반복)으로 실제 생성량 최소화.

## 2. 제작 순서 — 전체 워크플로우

### PHASE 0: 사전 준비 (Day 1)

```
[0-1] Kling AI 가입 + Standard 구독 ($6.99)
      → klingai.com 또는 fal.ai 가입

[0-2] Replicate 가입 + API 토큰 발급 (무료 시작)
      → replicate.com (SadTalker용)

[0-3] Suno AI에서 테스트곡 1곡 생성
      → 가사 + 프롬프트로 곡 품질 확인

[0-4] 예인 음성 준비 → PHASE 2.5에서 진행
      → Suno 곡 완성 후 보컬 추출 → GPT-SoVITS로 TTS 생성
      → greeting_voice.wav + break_voice.wav 2개만 만들면 끝
      → 이후 모든 영상에서 재사용 (초기 1회 셋팅)
```

### PHASE 1: 가사 작성 (Day 1~2)

```
[1-1] A세트 (예인의 이야기) 15곡 영어 가사 작성
      → Claude Code에서 작성 → lyrics/ 폴더에 저장

[1-2] B세트 (청자에게 전하는 이야기) 15곡 영어 가사 작성
      → 동일

[1-3] 가사 최종 검수
      → 한국인 발음에 자연스러운 단어 선택 확인
      → 복잡한 자음 클러스터 제거
```

### PHASE 2: 음악 생성 — Suno (Day 2~4)

```
[2-1] A세트 15곡 생성
      → Suno에 가사 + 공통 프롬프트 입력
      → 곡당 2~3회 생성 후 베스트 선별
      → music/vocals/A01_little_eyes.mp3 형태로 저장

[2-2] B세트 15곡 생성
      → 동일

[2-3] C세트 인스트루멘탈 8곡 생성
      → 인스트루멘탈 전용 프롬프트 사용
      → music/instrumental/ 에 저장

[2-4] 허밍 인트로 클립 생성 (선택)
      → 각 곡 시작부에 들어갈 5~10초 허밍
      → Suno에서 가사 없이 허밍만 생성
      → music/humming/ 에 저장

[2-5] 전체 오디오 품질 검수
      → 볼륨 레벨 일관성 확인
      → 톤/분위기 통일성 확인
```

### PHASE 2.5: 예인 음성 추출 + TTS 클론 (Day 3) — 초기 1회 셋팅

> **이 단계는 프로젝트 초기에 딱 1번만 하면 됩니다.**
> Suno 노래에서 보컬 추출 → GPT-SoVITS로 음성 클론 → 멘트 WAV 2개 생성.
> 이후 모든 영상(시즌 2, 3 포함)에서 동일 파일 반복 사용.
>
> 생성할 파일 (총 2개):
> - `greeting_voice.wav` — "안녕하세요, 예인입니다" (모든 영상 인트로)
> - `break_voice.wav` — "잠깐 쉬어갈게요" (모든 쉬는 시간 전환)

#### GPT-SoVITS 사용법 (추천 — MIT, 무료, 상업 OK)

```bash
# 설치
git clone https://github.com/RVC-Boss/GPT-SoVITS
cd GPT-SoVITS
pip install -r requirements.txt

# 1단계: 참조 오디오로 제로샷 TTS (5초 샘플만으로 가능)
python inference.py \
  --ref_audio ./예인_vocal_sample.wav \    # Suno에서 추출한 보컬
  --ref_text "I can feel the warmth" \     # 참조 오디오의 가사
  --target_text "안녕하세요 예인입니다" \    # 생성할 텍스트
  --output ./greeting_voice.wav

# 2단계: (선택) 1분 이상 샘플로 파인튜닝 → 더 정확한 음색 복제
```

#### Demucs 보컬 분리 (로컬, 무료, MIT)

```bash
# 설치
pip install demucs

# Suno 곡에서 보컬만 추출
demucs --two-stems=vocals music/vocals/A01_little_eyes.mp3
# → separated/htdemucs/A01_little_eyes/vocals.wav 생성
```

- 움직임 과할 때: "very subtle", "minimal movement" 추가
- 얼굴 변형 시: "maintaining exact same face and appearance" 추가
- 의상 변형 시: "wearing exact same all-black outfit" 추가

### PHASE 4: 코드 기반 영상 생성 (Day 4~5)

> PHASE 3과 **병렬 진행 가능**

```
[4-1] 각 보컬곡 × 노래 중 정지 영상 생성 (30곡)
      → scripts/build_demo_clean.py 엔진 활용
      → 포즈 이미지 + Breathing + Ken Burns + Warm Grade
      → 곡 길이(~3분)에 맞춰 영상 생성
      → video/segments/A01_singing.mp4 형태로 저장

[4-2] 인스트루멘탈 배경 영상 생성 (8곡)
      → 카페/스튜디오 배경 정지 이미지 + 미세 움직임
      → video/segments/C01_instrumental.mp4

[4-3] 곡 간 전환 효과 (크로스페이드 구간)
      → 허밍 클립 → 노래 정지 이미지 전환
      → 자연스러운 블렌딩
```

### PHASE 5: 자동 조립 — `assemble_full_video.py` (Day 5~6)

> **수동 편집 앱(CapCut 등) 사용하지 않음.**
> FFmpeg + Python 스크립트가 모든 소스를 자동으로 배치합니다.
> 소스 파일만 정해진 폴더에 넣으면 실행 한 번으로 2시간 영상 완성.
> VLLO(보유중)는 최종 미세 보정용으로만 사용.

#### 왜 FFmpeg 자동화인가 (vs 수동 편집앱)

- 38곡 × 동일 패턴 반복 → 코드가 1초, 사람이 하면 수 시간
- 시즌 2 제작 시 곡 파일만 교체하면 자동 재생성
- 2시간 영상을 iPad에서 편집하면 메모리/렌더링 문제
- CapCut 약관 리스크 없음 (콘텐츠 영구 사용권 부여 조항)
- 100% 무료, 상업적 제한 없음

#### 사용법 (3단계)

```
Step 1: 소스 파일을 정해진 폴더에 넣기
        → music/vocals/A01.mp3 ~ A15.mp3, B01.mp3 ~ B15.mp3
        → music/instrumental/C01.mp3 ~ C08.mp3
        → music/voice/greeting_voice.wav, break_voice.wav
        → video/clips/humming_a.mp4, humming_b.mp4, humming_c.mp4, ...

Step 2: 실행
        → python3 scripts/assemble_full_video.py

Step 3: 완성
        → video/output/yein_cafe_music_final.mp4 (2시간+)
        → video/output/timestamps.txt (유튜브 챕터용)
```

#### 스크립트가 자동으로 하는 일

```
1. 오디오 노멀라이즈 (전 곡 볼륨 통일)
2. 각 보컬곡에 포즈 이미지 매칭 → Breathing 영상 생성
3. 인스트루멘탈에 배경 이미지 매칭 → 정지 영상 생성
4. 타임라인 자동 배치:
   [인트로 클립] → [허밍A + A01 영상+음악] → [허밍B + A02] → ...
   → [멘트 + 퇴장] → [C01 인스트] → [C02 인스트] → [착석]
   → [허밍B + B01] → ... → [엔딩]
5. 클립 간 크로스페이드 (0.5초)
6. TTS 음성 삽입 (인사, 멘트)
7. 최종 H.264 렌더링 (1080p)
8. 타임스탬프 텍스트 자동 생성
```

#### 선택: VLLO로 후보정 (필요 시에만)

```
완성된 영상을 VLLO(iPad)로 열어서:
- 자막/가사 오버레이 추가
- 특정 구간 볼륨 미세 조정
- 엔드카드 디자인 추가
- 부분 트리밍
→ VLLO는 유료 보유 중이므로 상업적 문제 없음
```

```python
# pip install fal-client
import fal_client

result = fal_client.subscribe(
    "fal-ai/kling-video/v2.1/pro/image-to-video",
    arguments={
        "prompt": "[프롬프트]",
        "image_url": fal_client.upload_file("[이미지경로]"),
        "duration": "5",
        "aspect_ratio": "16:9",
    },
)
video_url = result["video"]["url"]
```

```python
# 직접 API 키 발급: https://klingai.com/global/dev/pricing
```

## 8. 주의사항 및 팁

### Kling 사용 시
- 무료 플랜 영상은 **절대 유튜브에 올리지 말 것** (워터마크 + 상업 불가)
- Standard 이상 구독 후 생성한 영상만 사용
- 생성 결과물에 대해 Kling에 영구 사용권이 부여됨 (약관상)
  → 예인 이미지가 Kling의 AI 학습/홍보에 쓰일 수 있음
  → 이게 싫으면 **Runway 사용** (생성물 소유권 사용자에게 귀속)

### SadTalker 사용 시
- Apache 2.0이므로 상업적 제한 없음
- 다만 결과물 품질이 Kling보다 떨어질 수 있음
- enhancer를 "gfpgan"으로 설정하면 얼굴 품질 향상

### 코드 효과 사용 시
- 자체 코드이므로 라이선스 문제 전혀 없음
- 곡 길이에 맞춰 영상 길이 조절 필요
- 포즈 이미지를 곡마다 다르게 배정하면 단조로움 방지

### 예인 이미지 자체의 저작권
- AI로 생성한 이미지라면 현행법상 저작권 보호가 불확실
- 원본 이미지의 생성 도구(Midjourney, DALL-E 등) 약관도 확인 필요
