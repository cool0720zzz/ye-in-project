## 3. 루프 영상 (Kaggle LTX) — 진행 중

- **케글 무료 T4가 루마보다 낫다(사용자 확정)**. 절차 문서: `무료GPU_루프영상_노트북.md`
- 노트북: kaggle.com/code/**linoailab/notebook058aeab969** (GPU T4×2·인터넷 ON 세팅 저장됨)
  - 셀1 = 자체완결 파이프라인: pip 설치 → LTX-Video 로드 → **base64 내장 앵커**(워커홀릭 키보드 스틸 512×320) → seed 1234 생성 → PSNR 루프컷 → `/kaggle/working/loop.mp4`
  - 검증된 시드 스캔 결과: 1234=31.95dB, 777=32.17, 2026=30.77, **4321=32.61 (BEST)** — 전부 "명확히 튐" 등급이라 **부메랑 폴백 필수**
  - 부메랑은 PC에서: `ffmpeg` 정방향+역방향 concat (loop_best.mp4 기반)
- **캐글→PC 파일 회수 (원격 검증된 방법)**: 다운로드 버튼·Output 패널·외부 업로드 전부 막힘 →
  **콘솔 base64 청크 릴레이**: `print('C1S'+B64[0:45000]+'C1E')` → `get_page_text`로 회수 → Write로 파트 저장 → PC에서 조립+MD5 대조 (`scripts/assemble_loop.py`).
  loop_best.mp4=167KB=b64 222,708자=**45K×5청크**. 콘솔 Clear 후 청크당 1회.
  - ⚠️ 45K 청크가 **말없이 일부 유실**될 수 있음(2026-07-24 part2가 3,202자 부족) → 조립 스크립트가 파트별 길이를 찍으니 **부족한 파트만 22.5K×2로 쪼개 재회수**(마커 D1S/D1E 등 새로 부여). 마커 3글자(C1E 등)는 base64 안에도 나올 수 있으니 **맨 끝 마커만** 제거.
  - ✅ 이 방법으로 A02 루프 회수 완료(MD5 e625970e... 일치). 부메랑은 캐글 것 대신 **PC ffmpeg**로: `split→reverse,trim=start_frame=1→concat`.
- ⚠️ **함정**: ① Run/Ctrl+Enter는 세션 부팅 중이거나 빈 셀 선택 시 조용히 무시됨 — 실행 확인은 "Cancel Run" 버튼 등장으로 ② 노트북 뷰가 빈 화면이면 새로고침 ③ **세션 유휴 리셋되면 /kaggle/working 파일 소실** — 완성 즉시 회수 ④ **GPU 주 30시간 쿼터 — 작업 끝나면 세션 전원 OFF 필수**

## 4. 갤러리 파이프라인

```
PYTHONIOENCODING=utf-8 python scripts/gallery_publish.py \
  --file <경로> --type image|loop|video|lyricvideo --id <고유id> --title "..." --verdict "..." [--final]
git add gallery/manifest.json && git commit && git pull --rebase && git push
```
- 같은 `--id` 재발행 = 카드 교체(리워크 반영에 사용)
- 썸네일은 manifest에 data URI 내장 → 소스 파일은 커밋 불필요(`grok_stills/`는 gitignore)
- 폰 결정은 `gallery/decisions*.json` 커밋으로 돌아옴 → **작업 전 git pull로 확인** (choice: continue/rework/hold + notes)
- push 전 `git pull --rebase` 습관화 (폰 결정 커밋과 자주 충돌)

- 프롬프트 = 꼬리표C(예인 인물, `video/그록_루프_프롬프트.md`) + 씬 + 꼬리표A(실사 영상미), **16:9·이미지 모드·품질**

- 전송 성공 판정 = **탭 타이틀이 프롬프트로 바뀜**. 실패 시 입력창 재클릭→타이핑→↑버튼
- 다운로드는 **상세 페이지 완전 로드 후** 우하단 아이콘. 파일명 `grok-image-<id>.jpg` 또는 `<id>.jpg`
- 발행된 스틸 원본: `grok_stills/` (A02 키보드, A03 배시시, A04 인파, A05 빨간불, B01 전화)

## 5-1. Suno 조작 워크플로우 (브라우저 원격, 검증됨)

- **곡 생성 순서(Advanced)**: `+Voice`로 페르소나 부착 → "Overwrite Styles?" 뜨면 스타일 이미 넣었을 때 **Keep Current** → 가사 → Styles → Song Title → Create.
- **가사칸은 contenteditable** — form_input 안 됨. 클릭 후 타이핑. ⚠️ 긴 한글 가사 일괄 타이핑 시 렌더러 20~30초 프리즈 → **정상, 입력 큐 빠지면 복구**. 기다렸다 스크린샷으로 확인.
- Styles/Title 칸은 form_input 됨(ref_44/ref_252 패턴).
- **페르소나 만들기**: 라이브러리 클립 `⋯ → Remix → Voice(Pro)` → "Create a Voice" → 파형에서 **후렴 구간** 선택(타임은 `music/<곡>/<곡>.lrc` 참조) → 이름 → **Public OFF(비공개)** → Save. 저장하면 Create 패널에 자동 부착.
- **같은 곡 다른 페르소나 A/B (핵심 트릭)**: 기존 클립 `⋯ → Remix → Reuse Prompt` → 가사·스타일·제목 로드됨 → 페르소나만 교체 → 제목에 구분자 → Create. 가사 재입력(프리즈) 없이 안전.
- 페르소나 카드에서 썸네일 중앙 클릭 = **미리듣기 재생**(선택 아님) — **이름 텍스트를 클릭**해야 선택됨.

## 5-2. 폰 원격(RC) 운영 원칙 — 사용자와 합의됨

1. **폰 사용자는 PC 화면을 못 본다** — 결과는 설명 말고 **갤러리 발행/파일 전송**으로. ("대답", "화면 띄워" 반복 = 결과가 안 닿는다는 신호)
2. **RC에선 AskUserQuestion 메뉴가 안 보인다** — 질문은 일반 텍스트 번호 선택지(1/2/3)로.
3. **연결 끊김**: 작업은 계속. 되돌릴 수 있는 결정은 안전 기본값으로 진행 후 보고, 비가역·대량비용·공개 액션은 대기. 재접속 첫 메시지는 상태 요약.
4. 폰 채팅이 실시간 갱신 안 되고 새로고침해야 보이는 건 RC 스트림 특성 — 정상.
5. PC↔폰 공유 채널은 **git / OneDrive / 웹서비스 계정** 뿐. PC 세션에 시킨 웹작업은 계정에서 직접 확인.
6. Claude in Chrome은 자기 MCP 탭 그룹만 제어 — 사용자가 켜둔 탭 접근 불가, **URL을 받아** 새 탭으로 여는 게 빠름.
7. 상세 표준: 로컬 스킬 `~/.claude/skills/ye-in-phone-remote-ops/SKILL.md` (이 PC 한정).
